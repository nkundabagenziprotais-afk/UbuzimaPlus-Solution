# PharmaCo360 Operations Command Center Checksum Register

## Purpose

This register records checksum evidence for a future approved PharmaCo360 operations command center production package.

This register does not generate a checksum by itself, create a package, upload a package, or deploy to production. It defines the evidence fields required when an approved package is created later.

## Register identity

Product: PharmaCo360  
Area: Operations command center  
Evidence type: Production package checksum register  
Source branch: main  
Baseline before Phase 15.4: 9f7d4ca  

## Checksum register principle

Every approved production package must have checksum evidence that allows the deployment operator and reviewer to confirm package integrity before deployment.

## Required checksum register fields

Record:

- release phase
- package filename
- package location
- approved source commit
- source branch
- package creation date
- package created by
- package size
- checksum method
- checksum value
- checksum generated by
- checksum verification date
- checksum verified by
- manifest reference
- rollback reference
- deployment handoff reference
- approval reference
- final decision

## Approved checksum methods

Use one approved method:

- SHA256
- SHA512

## Checksum generation rule

Checksum must be generated only after the final package file has been created.

If the package contents change, the checksum must be regenerated and the old value must not be used for deployment approval.

## Verification rule

Before deployment, the deployment operator must confirm:

- received package filename matches register
- received package size matches register
- received checksum matches register
- source commit matches approved main
- package manifest is complete
- protected-file inspection passed
- rollback reference exists

## Register pass criteria

Checksum register passes when:

- package filename is recorded
- package location is recorded
- source commit is recorded
- package size is recorded
- checksum method is recorded
- checksum value is recorded
- manifest reference is recorded
- rollback reference is recorded
- reviewer is identified
- final decision is recorded separately

## Register fail criteria

Checksum register fails when:

- package filename is missing
- package location is missing
- source commit is missing
- checksum method is missing
- checksum value is missing
- package changed after checksum generation
- manifest reference is missing
- rollback reference is missing
- reviewer is missing
