# PharmaCo360 Operations Command Center Checksum Evidence Checklist

## Purpose

This checklist defines checksum evidence required for the PharmaCo360 operations command center production package.

It does not create, copy, upload, or deploy any production package. It only defines the checksum control evidence required before deployment approval.

## Checksum identity

Product: PharmaCo360  
Area: Operations command center  
Evidence type: Package checksum and integrity control  
Source branch: main  
Baseline before Phase 15.2: f453e81  

## Checksum principle

A production package must have checksum evidence so the deployment operator can confirm the package received is the same package approved for deployment.

## Required checksum evidence

Record:

- package filename
- package location
- checksum method
- checksum value
- checksum generation date
- checksum generated by
- source commit
- reviewer
- verification result
- handoff recipient

## Recommended checksum methods

Use one approved method and record it:

- SHA256
- SHA512

## Required verification steps

Before handoff, confirm:

- package exists in the approved package location
- checksum is generated after final package creation
- checksum value is copied into the evidence register
- checksum is not generated before package contents are finalized
- checksum is verified by reviewer or deployment operator
- checksum evidence is linked to the package manifest

## Integrity verification

Deployment operator should confirm:

- received package filename matches manifest
- received package size matches manifest
- received package checksum matches manifest
- received package source commit matches approved main
- received package does not include protected files

## Protected-file checksum caution

Do not generate approval evidence for a package that contains:

- .env
- .env backup files
- local database files
- local cache files
- logs
- node_modules unless explicitly approved
- production secrets
- production user data

## Checksum pass criteria

Checksum evidence passes when:

- package filename is recorded
- checksum method is recorded
- checksum value is recorded
- source commit is recorded
- verification result is recorded
- package manifest reference is recorded
- reviewer is identified

## Checksum fail criteria

Checksum evidence fails when:

- checksum is missing
- checksum method is unclear
- package filename is unclear
- source commit is unclear
- package contents changed after checksum generation
- protected files are included
- reviewer is missing
